# MiniLangVM: семантика языка

## 1. Что такое семантика

Грамматика отвечает за то, как программа выглядит.

А семантика отвечает за то, имеет ли программа смысл.

Например синтаксически можно написать:

```
int x = "hello";
```

Но по смыслу это ошибка, потому что переменная типа `int`, а справа строка.

Именно такие вещи проверяет semantic analyzer.

Общий путь программы такой:

```
исходный код
    -> lexer
    -> parser
    -> AST
    -> semantic analyzer
    -> bytecode generator
    -> VM
```

То есть semantic analyzer работает уже после парсера, когда AST построен.

---

## 2. Что разрешено на верхнем уровне

На верхнем уровне программы можно писать только объявления:

```
функции
структуры
type aliases
namespace
```

Нельзя писать обычные инструкции прямо в файле.

Нельзя так:

```
int x = 10;
print(x);
```

Нужно так:

```
int main() {
    int x = 10;
    print(x, "\n");
    return 0;
}
```

Точка входа в программу:

```
int main()
```

`main` должен:

```
существовать
возвращать int
не иметь параметров
```

Возвращаемое значение `main` становится кодом завершения программы.

---

## 3. Стратегия вычисления

В MiniLangVM используется call-by-value.

Это значит, что аргументы функции сначала вычисляются, а потом передаются в функцию как значения.

Пример:

```
int add(int a, int b) {
    return a + b;
}

int main() {
    int x = add(10, 20);
    return x;
}
```

Сначала вычисляется `10`, потом `20`, потом вызывается `add`.

Параметры `a` и `b` внутри функции — это локальные переменные функции.

Если внутри функции изменить параметр, исходная переменная снаружи не должна меняться напрямую.

---

## 4. Области видимости

В языке используется lexical scoping.

То есть переменная видна в том блоке, где она объявлена, и во вложенных блоках.

Пример:

```
int main() {
    int x = 10;

    {
        print(x, "\n");
    }

    return 0;
}
```

Здесь внутренний блок видит `x`.

Каждый блок `{ ... }` создаёт новую область видимости.

Можно делать shadowing, то есть внутренняя переменная может скрыть внешнюю:

```
int main() {
    int x = 10;

    {
        int x = 20;
        print(x, "\n");
    }

    print(x, "\n");

    return 0;
}
```

Внутри блока напечатается 20, снаружи 10.

Но в одной и той же области нельзя объявить две переменные с одним именем.

Так нельзя:

```
int main() {
    int x = 10;
    int x = 20;
    return 0;
}
```

Использовать переменную до объявления тоже нельзя.

---

## 5. Объявление переменных

Переменную можно объявить с явным типом:

```
int x = 10;
string s = "hello";
bool ok = true;
```

Можно использовать `var`:

```
var x = 10;
var s = "hello";
```

`var` означает, что тип выводится по инициализатору.

Например:

```
var x = 10;
```

Компилятор понимает, что `x` имеет тип `int`.

После этого `x` становится обычной статически типизированной переменной. То есть нельзя потом положить туда строку:

```
var x = 10;
x = "hello";
```

Это ошибка.

В моём языке переменная должна иметь начальное значение. Просто объявить без инициализации нельзя.

---

## 6. let и мутабельность

Обычные переменные изменяемые:

```
int x = 10;
x = 20;
```

`var` тоже создаёт изменяемую переменную:

```
var x = 10;
x = 20;
```

А `let` создаёт неизменяемую переменную:

```
let x = 10;
```

После этого нельзя писать:

```
x = 20;
```

Это semantic error.

Важно: мутабельность — это свойство переменной, а не типа.

То есть `int` сам по себе не “изменяемый” и не “неизменяемый”. Изменяемость зависит от того, как объявлена переменная.

---

## 7. Type aliases

`type alias` даёт другое имя существующему типу.

Пример:

```
type Meters = int;

int main() {
    Meters distance = 100;
    int x = distance;
    return x;
}
```

`Meters` — это не новый отдельный тип.

Это просто другое имя для `int`.

Поэтому `Meters` и `int` совместимы.

---

## 8. Структуры

Структура — это пользовательский тип с полями.

Пример:

```
struct Point {
    int x;
    int y;
}
```

Создание структуры:

```
Point p = Point { x: 10, y: 20 };
```

Доступ к полю:

```
print(p.x, "\n");
```

Изменение поля:

```
p.x = 30;
```

Структуры имеют nominal typing.

Это значит, что тип определяется именем структуры.

Например если есть две структуры с одинаковыми полями:

```
struct A {
    int x;
}

struct B {
    int x;
}
```

то `A` и `B` всё равно разные типы.

При создании struct literal проверяется:

```
все поля указаны
нет лишних полей
нет повторов одного поля
тип каждого значения подходит к типу поля
```

Например так нельзя:

```
Point p = Point { x: 10 };
```

Потому что не указано поле `y`.

Сравнение структур:

```
p == q
p != q
```

работает по полям.

---

## 9. Массивы

Массивы имеют фиксированный размер.

Пример:

```
int[3] a = [1, 2, 3];
```

Размер является частью типа.

То есть:

```
int[3]
```

и

```
int[4]
```

это разные типы.

Литерал массива должен иметь правильный размер.

Так нельзя:

```
int[3] a = [1, 2];
```

Потому что тип требует 3 элемента, а в литерале только 2.

Элементы массива должны быть совместимых типов.

Доступ к элементу:

```
a[0]
```

Присваивание элементу:

```
a[1] = 10;
```

Длина массива:

```
len(a)
```

Индекс должен быть типа `int`.

Если индекс выходит за границы, это runtime error:

```
runtime error: array index out of bounds at line N
```

---

## 10. Массивы разных типов

После доработок массивы могут хранить не только `int`.

Поддерживаются массивы:

```
int
uint
float64
bool
string
struct
```

Примеры:

```
uint[3] flags = [cast<uint>(1), cast<uint>(2), cast<uint>(4)];
float64[2] limits = [0.0, 100.0];
bool[2] checks = [true, false];
string[2] names = ["Almas", "Bob"];
```

Массив структур:

```
struct Point {
    int x;
    int y;
}

Point[2] points = [
    Point { x: 1, y: 2 },
    Point { x: 3, y: 4 }
];
```

Массивы можно сравнивать:

```
a == b
a != b
```

Сравнение идёт поэлементно.

---

## 11. Массивы внутри структур

Структура может иметь поле-массив:

```
struct Group {
    string name;
    int[3] grades;
    string[2] students;
}
```

Пример:

```
Group g = Group {
    name: "VM group",
    grades: [90, 80, 70],
    students: ["Almas", "Bob"]
};
```

Можно обращаться к элементу массива внутри структуры:

```
print(g.grades[0], "\n");
```

Можно изменять элемент:

```
g.grades[1] = 100;
```

Это важная часть проекта, потому что здесь вместе работают структуры, массивы, доступ к полям, индексирование и присваивание.

---

## 12. Функции

Функция имеет:

```
имя
список параметров
тип возвращаемого значения
тело
```

Пример:

```
int add(int a, int b) {
    return a + b;
}
```

Вызов:

```
add(10, 20)
```

Количество аргументов должно совпадать с количеством параметров.

Типы аргументов должны подходить к типам параметров.

Если функция возвращает `int`, она должна вернуть `int`.

Если функция `void` или `unit`, она может писать:

```
return;
```

Но не может вернуть значение.

Так нельзя:

```
void f() {
    return 10;
}
```

И так нельзя:

```
int f() {
    return;
}
```

---

## 13. Generic-функции

В MiniLangVM есть generic-функции.

Пример:

```
T id<T>(T x) {
    return x;
}
```

`T` — это параметр типа.

Его можно указать явно:

```
int a = id<int>(42);
```

А можно не указывать, тогда компилятор выведет тип сам:

```
string s = id("hello");
```

Здесь компилятор понимает, что `T = string`.

Правила:

```
generic-параметр должен быть объявлен в <T>
один и тот же T должен означать один и тот же тип в одном вызове
если тип нельзя вывести, будет ошибка
main не может быть generic-функцией
```

Пример ошибки:

```
T choose<T>(bool cond, T a, T b) {
    if (cond) {
        return a;
    }

    return b;
}

int main() {
    int x = choose(true, 10, "bad");
    return x;
}
```

Тут `T` одновременно получается `int` и `string`, поэтому вызов неправильный.

Generic structs у меня не реализованы.

Реализация generic ближе к type erasure: типы проверяются на этапе semantic analysis, а во время выполнения VM работает с обычным `Value`.

---

## 14. Перегрузка функций

В проекте есть перегрузка функций.

Можно объявить несколько функций с одним именем, если параметры разные:

```
int describe(int x) {
    return 1;
}

int describe(string x) {
    return 2;
}
```

Вызов:

```
describe(10)
```

выберет функцию для `int`.

Вызов:

```
describe("hello")
```

выберет функцию для `string`.

Если две функции имеют одинаковую сигнатуру, это ошибка:

```
int f(int x) {
    return 1;
}

int f(int x) {
    return 2;
}
```

Semantic analyzer хранит список функций с одним именем и при вызове выбирает подходящую по типам аргументов.

После выбора в AST записывается конкретный functionIndex, поэтому codegen уже не ищет функцию заново.

---

## 15. Перегрузка с неявными приведениями

Также есть перегрузка с неявными числовыми приведениями.

Если точной перегрузки нет, компилятор пробует безопасные приведения:

```
int  -> float64
uint -> float64
```

Пример:

```
int onlyFloat(float64 x) {
    return 10;
}

int main() {
    return onlyFloat(42);
}
```

Тут `42` имеет тип `int`, а функция принимает `float64`.

Точной функции `onlyFloat(int)` нет, поэтому компилятор выбирает `onlyFloat(float64)` и вставляет приведение.

Но точное совпадение всегда важнее.

Пример:

```
int f(int x) {
    return 1;
}

int f(float64 x) {
    return 2;
}

int main() {
    return f(42);
}
```

Здесь выберется `f(int)`, а не `f(float64)`.

---

## 16. Управление потоком

### Блок

Блок:

```
{
    int x = 10;
    print(x, "\n");
}
```

создаёт новую область видимости.

### Пустая инструкция

Пустая инструкция:

```
;
```

ничего не делает, но синтаксически допустима.

### if / else

Условие в `if` должно быть типа `bool`.

```
if (x > 0) {
    print("positive\n");
} else {
    print("non-positive\n");
}
```

Если условие не bool, будет semantic error.

### while

Условие в `while` тоже должно быть bool.

```
while (i < 10) {
    i = i + 1;
}
```

Тело выполняется, пока условие true.

### for

`for` состоит из инициализации, условия и обновления:

```
for (int i = 0; i < 10; i = i + 1) {
    print(i, "\n");
}
```

По смыслу это похоже на:

```
int i = 0;

while (i < 10) {
    print(i, "\n");
    i = i + 1;
}
```

### switch

`switch` выбирает ветку по значению:

```
switch (x) {
    case 1:
        print("one\n");
        break;

    default:
        print("other\n");
        break;
}
```

case-значения должны быть совместимы с типом switch-выражения.

### break

`break` можно использовать внутри цикла или switch.

Он завершает ближайший цикл или switch.

### continue

`continue` можно использовать только внутри цикла.

Он переходит к следующей итерации.

### return

`return` завершает текущую функцию.

В функции с типом возврата надо вернуть значение нужного типа.

В `void` / `unit` функции можно писать:

```
return;
```

## 17. Арифметика

Поддерживаются операции:

```
+
-
*
/
%
```

Для `int` результат `int`.

Для `uint` результат `uint`.

Для `float64` поддерживаются:

```
+
-
*
/
```

Остаток `%` для `float64` не поддерживается.

Деление на ноль — runtime error.

Это касается и `int`, и `uint`, и `float64`.

То есть у меня `1.0 / 0.0` не даёт `inf`, а даёт ошибку времени выполнения.

Специальные значения float пишутся как литералы:

```
inf
-inf
NaN
```

---

## 18. Строки

Для строк поддерживается конкатенация через `+`:

```
string s = "hello" + " world";
```

Сравнение:

```
s == "hello"
s != "test"
```

Длина:

```
len(s)
```

Отдельного типа `char` нет. Один символ можно хранить как строку длины 1.

---

## 19. Сравнения

Для чисел доступны:

```
==
!=
<
<=
>
>=
```

Для bool:

```
==
!=
```

Для string:

```
==
!=
```

Для массивов:

```
==
!=
```

Для структур:

```
==
!=
```

Результат сравнения всегда `bool`.

---

## 20. Логические операции

Поддерживаются:

```
!
&&
||
```

Операнды должны быть `bool`.

Пример:

```
if (ok && ready) {
    print("yes\n");
}
```

Результат тоже `bool`.

---

## 21. Битовые операции

Есть битовые операции:

```
&
|
^
~
<<
>>
```

Они работают для `int` и `uint`.

Для `&`, `|`, `^` типы должны совпадать:

```
int & int
uint & uint
```

Так нельзя:

```
int x = 1;
uint y = cast<uint>(2);
x & y;
```

Нужно явно привести типы.

`~` работает для `int` и `uint`.

Сдвиги:

```
x << n
x >> n
```

Левый операнд должен быть `int` или `uint`.

Правый операнд тоже должен быть `int` или `uint`.

Если сдвиг отрицательный или `>= 32`, будет runtime error:

```
runtime error: invalid shift count at line N
```

---

## 22. Инкремент, декремент и составное присваивание

Есть:

```
x++;
x--;
```

Есть:

```
x += expr;
x -= expr;
```

Они работают с изменяемыми числовыми переменными.

Для `let` это запрещено.

То есть так нельзя:

```
let x = 10;
x++;
```

Потому что `x` неизменяемый.

---

## 23. cast

Явное приведение типа:

```
cast<int>(x)
cast<float64>(x)
cast<uint>(x)
```

Примеры:

```
int x = cast<int>(3.5);
float64 y = cast<float64>(x);
uint u = cast<uint>(10);
```

Недопустимый cast должен давать semantic error.

## 24. Builtins

### print

`print` выводит значения в stdout.

Можно передавать несколько аргументов:

```
print("x = ", x, "\n");
```

Поддерживаемые типы:

```
int
uint
float64
bool
string
arrays
structs
```

### input

`input()` читает строку из stdin.

```
string s = input();
```

Аргументов не принимает.

Возвращает `string`.

### len

`len` возвращает длину строки или массива.

```
len("abc")
len(a)
```

Результат — `int`.

### exit

`exit(code)` завершает программу с указанным кодом.

```
exit(1);
```

Аргумент должен быть `int`.

### panic

`panic(message)` завершает программу с runtime error.

```
panic("something went wrong");
```

Аргумент должен быть `string`.

### assert

`assert` проверяет условие:

```
assert(x > 0);
assert(x > 0, "x must be positive");
```

Если условие false, программа завершается с runtime error.

Первый аргумент должен быть `bool`.

Второй аргумент, если есть, должен быть `string`.

### sizeof

`sizeof(expr)` возвращает размер типа выражения в байтах.

Примеры:

```
sizeof(42)
sizeof(2.5)
sizeof(true)
sizeof("hello")
```

Для массивов учитывается количество элементов.

### typeid

`typeid(expr)` возвращает строку с типом выражения.

Пример:

```
typeid(42)
```

вернёт что-то вроде:

```
int
```

### typeof

`typeof(expr)` сейчас по смыслу похож на `typeid`.

Он тоже возвращает строковое представление статического типа выражения.

---

## 25. Ошибки компиляции

Ошибки лексера, парсера и семантики выводятся в таком формате:

```
file:line:column: error: message
```

Пример:

```
examples/test.ml:3:10: error: cannot assign string to int
```

Компилятор останавливается на первой ошибке текущей фазы.

Для ошибок используется `Result<T>` и `Diagnostic`.

Исключения не являются основным механизмом диагностики.

---

## 26. Runtime errors

Runtime errors возникают уже во время выполнения VM.

Формат примерно такой:

```
runtime error: message at line N
```

Примеры:

```
division by zero
modulo by zero
array index out of bounds
invalid shift count
assert failure
panic
invalid VM state
```

После runtime error программа завершается с ненулевым кодом.

---

## 27. Память

В языке нет ручного управления памятью.

Нет `malloc`, `free`, указателей и адресной арифметики.

Строки, массивы и структуры хранятся как значения VM.

Внутри реализации C++ могут использоваться runtime-структуры и умные указатели, но пользователь языка этим напрямую не управляет.

Для пользователя всё выглядит как value semantics.

## 28. Что пока не реализовано

Не реализовано:

```
отдельный тип char
generic structs
pointers
nullable types
lambdas
first-class functions
overloaded operators
полноценная модульная система
IR-level optimizations
```

Перегрузка функций уже реализована, поэтому её не надо писать в ограничениях.

Основной backend — bytecode VM.

x86-64 ASM backend есть, но он дополнительный и поддерживает только часть языка.

Semantic analyzer проверяет смысл программы после построения AST. Он проверяет области видимости, объявления переменных, типы выражений, return, if/while/for условия, массивы, структуры, generic-функции, перегрузку функций и встроенные функции.

В языке строгая статическая типизация. Если типы не совпадают, ошибка появляется до генерации байткода. Runtime-ошибки остаются только для ситуаций, которые зависят от значения во время выполнения: деление на ноль, выход массива за границы, неправильный сдвиг, assert и panic.

Основная идея такая: parser строит дерево, semantic analyzer проверяет дерево и дописывает туда типы и индексы функций, после этого codegen уже может спокойно генерировать bytecode.
