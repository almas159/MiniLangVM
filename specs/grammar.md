# Грамматика MiniLangVM

## 1. Общая идея

MiniLangVM использует синтаксис, похожий на C / Java.

То есть программа выглядит примерно так:

```
int main() {
    int x = 10;
    print(x, "\n");
    return 0;
}
```

На верхнем уровне программы можно объявлять:

```
функции
структуры
синонимы типов
namespace
```

Обычные инструкции типа `if`, `while`, `return`, `print` могут быть только внутри функций.

Точка входа в программу:

```
int main()
```

Функция `main` должна возвращать `int`.

---

## 2. Лексика

Лексер разбивает текст программы на токены.

Токены — это ключевые слова, идентификаторы, числа, строки, операторы и скобки.

Например строка:

```
int x = 10;
```

разбивается примерно так:

```
int
x
=
10
;
```

---

## 3. Пробелы и переносы строк

Пробелы, табы и переносы строк нужны только чтобы разделять токены.

Они не попадают в AST и не влияют на смысл программы.

Например эти две записи одинаковые:

```
int x = 10;
```

и

```
int
x
=
10
;
```

---

## 4. Комментарии

Есть однострочные комментарии:

```
// это комментарий
```

Есть блочные комментарии:

```
/*
   это многострочный комментарий
*/
```

Комментарии просто пропускаются лексером.

То есть для парсера и семантики их как будто нет.

---

## 5. Идентификаторы

Идентификатор — это имя переменной, функции, структуры, поля и т.д.

Примеры:

```
x
sum
main
Point
studentName
```

Идентификатор может состоять из букв, цифр и `_`.

Но начинаться с цифры нельзя.

Правильно:

```
value1
my_var
Point
```

Неправильно:

```
1value
```

Ключевые слова нельзя использовать как имена переменных.

Например нельзя назвать переменную `int` или `return`.

## 8. Вещественные литералы

Вещественные числа:

```

1.0
```

Также есть запись с экспонентой:

```

```

Это значит:

```

2.5E1 = 25
```

Есть специальные значения:

```

NaN

```

Например:

```

```

Важно: в моей VM деление на ноль даёт runtime error, даже для float64. Поэтому специальные значения лучше получать именно через литералы `inf`, `-inf`, `NaN`, а не через `1.0 / 0.0`.

## 11. Общая структура программы

Программа состоит из объявлений верхнего уровня:

```
Program ::= TopLevelDecl*
```

На верхнем уровне могут быть:

```
FunctionDecl
StructDecl
TypeAliasDecl
NamespaceDecl
```

То есть на верхнем уровне нельзя писать так:

```
int x = 10;
```

Это должно быть внутри функции.

Правильно:

```
int main() {
    int x = 10;
    return x;
}
```

---

## 12. Namespace

Есть namespace:

```
namespace Math {
    int square(int x) {
        return x * x;
    }
}
```

Функции внутри namespace получают квалифицированное имя.

Вызов выглядит так:

```
Math::square(5)
```

namespace нужен, чтобы группировать функции и не смешивать имена.

---

## 13. Синонимы типов

Можно объявлять alias типа:

```
type Meters = int32;
```

После этого можно писать:

```
Meters distance = 100;
```

Но это не новый отдельный тип, а просто другое имя для уже существующего типа.

То есть:

```
Meters
```

по смыслу то же самое, что:

```
int32
```

## 14. Структуры

Структура объявляет свой тип с набором полей.

Пример:

```
struct Point {
    int x;
    int y;
}
```

Потом можно создать значение:

```
Point p = Point { x: 10, y: 20 };
```

Доступ к полю:

```
print(p.x, "\n");
```

Изменение поля:

```
p.x = 30;
```

Структуры в моём языке имеют nominal typing.

То есть две структуры с одинаковыми полями, но разными именами — это разные типы.

## 16. Функции

Обычная функция:

```
int add(int a, int b) {
    return a + b;
}
```

Функция состоит из:

```
тип возвращаемого значения
имя
параметры
тело
```

Пример:

```
int main() {
    return 0;
}
```

Если функция ничего не возвращает, используется `void` или `unit`:

```
void hello() {
    print("hello\n");
    return;
}
```

---

## 17. Generic-функции

Есть обобщённые функции.

Пример:

```
T identity<T>(T x) {
    return x;
}
```

Её можно вызвать явно:

```
int a = identity<int>(42);
```

А можно с автовыводом типа:

```
string s = identity("hello");
```

В этом случае компилятор сам понимает, что `T = string`.

Generic-параметр нужен на этапе проверки типов. Во время выполнения VM работает с обычным Value.

Моя реализация generic ближе к type erasure, то есть отдельная копия функции под каждый тип не создаётся.

---

## 18. Перегрузка функций

Можно объявить несколько функций с одним именем, если у них разные параметры:

```
int describe(int x) {
    return 1;
}

int describe(string x) {
    return 2;
}
```

Вызов:

```
describe(10)
```

выберет первую функцию.

Вызов:

```
describe("hello")
```

выберет вторую функцию.

Если две функции имеют одинаковую сигнатуру, это ошибка:

```
int f(int x) {
    return 1;
}

int f(int x) {
    return 2;
}
```

Так нельзя, потому что компилятор не поймёт, какую именно функцию вызвать.

---

## 19. Перегрузка с неявным приведением

Если точной перегрузки нет, компилятор может попробовать безопасное числовое приведение.

В моём проекте для перегрузки есть такие приведения:

```
int  -> float64
uint -> float64
```

Пример:

```
int onlyFloat(float64 x) {
    return 10;
}

int main() {
    return onlyFloat(42);
}
```

Тут `42` имеет тип `int`, а функция принимает `float64`.

Компилятор сам вставляет приведение `int -> float64`.

Но если есть точное совпадение, оно важнее.

Например:

```
int f(int x) {
    return 1;
}

int f(float64 x) {
    return 2;
}

int main() {
    return f(42);
}
```

Выберется `f(int)`, а не `f(float64)`.

## 21. Объявление переменной

```

```

```
var x = 10;
```

Неизменяемая переменная:

```

```

```

## 23. Массивы

Массив имеет фиксированный размер.

Пример:

```
int[3] a = [1, 2, 3];
```

Индексирование:

```
print(a[0], "\n");
```

Изменение элемента:

```
a[1] = 10;
```

Длина массива:

```
len(a)
```

Если индекс выходит за границы, будет runtime error.

Например:

```
int[2] a = [10, 20];
print(a[2]);
```

Индекс 2 неправильный, потому что есть только 0 и 1.

## 24. Массивы разных типов

После доработок массивы могут быть не только `int`.

Например:

```
uint[3] flags = [cast<uint>(1), cast<uint>(2), cast<uint>(4)];
float64[2] limits = [0.0, 100.0];
bool[2] checks = [true, false];
string[2] names = ["Almas", "Bob"];
```

Также можно делать массивы структур:

```
struct Point {
    int x;
    int y;
}

Point[2] points = [
    Point { x: 1, y: 2 },
    Point { x: 3, y: 4 }
];
```

---

## 25. Массивы внутри структур

Структура может иметь поле-массив:

```
struct Group {
    string title;
    int[3] grades;
    string[2] names;
}
```

Пример использования:

```
Group g = Group {
    title: "VM group",
    grades: [90, 80, 70],
    names: ["Almas", "Bob"]
};

print(g.grades[0], "\n");
g.grades[1] = 100;
```

У меня поддержано индексирование массивов, которые лежат в полях структуры:

```
g.grades[0]
g.names[1]
```

Но цепочка вида:

```
g.points[0].x
```

может не поддерживаться напрямую. В таком случае надо сначала достать элемент массива в переменную:

```
Point p = g.points[0];
print(p.x, "\n");
```

## 27. if / else

Условный оператор:

```
if (x > 0) {
    print("positive\n");
} else {
    print("non-positive\n");
}
```

Условие должно иметь тип `bool`.

Если условие не bool, semantic analyzer выдаст ошибку.

## 28. while

Цикл while:

```
int i = 0;

while (i < 10) {
    print(i, "\n");
    i = i + 1;
}
```

Сначала проверяется условие.

Если оно true — выполняется тело.

Если false — цикл заканчивается.

---

## 29. for

Цикл for:

```
for (int i = 0; i < 10; i = i + 1) {
    print(i, "\n");
}
```

Он состоит из:

```
инициализации
условия
обновления
тела
```

По смыслу это похоже на while:

```
int i = 0;

while (i < 10) {
    print(i, "\n");
    i = i + 1;
}
```

---

## 30. break и continue

`break` выходит из цикла:

```
while (true) {
    break;
}
```

`continue` переходит к следующей итерации:

```
while (i < 10) {
    i = i + 1;

    if (i == 5) {
        continue;
    }

    print(i, "\n");
}
```

`continue` можно использовать только внутри циклов.

`break` можно использовать в циклах и switch.

---

## 31. switch

Есть switch:

```
switch (x) {
    case 1:
        print("one\n");
        break;

    case 2:
        print("two\n");
        break;

    default:
        print("other\n");
        break;
}
```

switch выбирает ветку по значению выражения.

case задаёт вариант.

default выполняется, если ничего не подошло.

---

## 32. return

В функции с возвращаемым типом надо вернуть значение:

```
int f() {
    return 10;
}
```

В `void` / `unit` функции можно писать return без выражения:

```
void hello() {
    print("hello\n");
    return;
}
```

Если функция должна вернуть `int`, а return пустой — это ошибка.

## 34. Выражения и приоритеты

Выражения разбираются по приоритетам.

От низкого приоритета к высокому:

```
||
&&
|
^
&
== !=
< <= > >=
<< >>
+ -
* / %
унарные операции
primary
```

То есть:

```
2 + 3 * 4
```

понимается как:

```
2 + (3 * 4)
```

а не как:

```
(2 + 3) * 4
```

## 39. Литералы массивов

Массив создаётся через квадратные скобки:

```
[1, 2, 3]
```

Но тип массива обычно задаётся в объявлении:

```
int[3] a = [1, 2, 3];
```

Количество элементов должно совпадать с размером типа.

Так нельзя:

```
int[3] a = [1, 2];
```

Потому что в типе размер 3, а элементов только 2.

## 40. Литералы структур

Пример:

```
struct Student {
    string name;
    int score;
}

Student s = Student {
    name: "Almas",
    score: 95
};
```

Если указать неправильное поле, semantic analyzer выдаст ошибку.

## 41. Встроенные функции

Есть встроенные функции:

```
input()
len(expr)
exit(expr)
panic(expr)
assert(condition)
assert(condition, message)
sizeof(expr)
typeid(expr)
typeof(expr)
```

`input()` читает строку.

`len()` работает для строк и массивов.

`exit()` завершает программу.

`panic()` завершает программу с ошибкой.

`assert()` проверяет условие.

`sizeof()` возвращает размер значения или типа в байтах.

`typeid()` возвращает строковое имя типа.

`typeof()` возвращает строковое представление типа выражения.

    int y;

int main() {
    Point p = Point { x: 10, y: 20 };

    print(p.x, "\n");

    p.x = 30;

    print(p.x, "\n");

    return 0;
}

Грамматика языка C-подобная. Верхний уровень состоит из объявлений функций, структур, type aliases и namespace. Инструкции находятся внутри функций. Есть строгая система типов, массивы фиксированного размера, структуры, generic-функции, перегрузка функций, bitwise-операции, if/while/for/switch, встроенные функции и собственная VM.

Парсер строит AST, а уже semantic analyzer проверяет типы, существование переменных, правильность return, корректность индексов массивов по типу, generic-вызовы и перегрузки функций. Поэтому грамматика отвечает только за синтаксис, а правила типов проверяются позже.
