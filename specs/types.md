# MiniLangVM: система типов

## 1. Общая идея

В MiniLangVM статическая строгая система типов.

Это значит, что типы проверяются до выполнения программы, на этапе semantic analysis.

Например так нельзя:

```
int x = "hello";
```

Потому что слева `int`, а справа `string`.

Компилятор должен выдать ошибку примерно такого вида:

```
file:line:column: error: message
```

То есть ошибка находится ещё до генерации байткода и до запуска VM.

---

## 2. Какие типы есть

В языке есть такие основные типы:

```
int
int32
uint
uint32
float64
float32
bool
string
void
unit
arrays
structs
generic type parameters
```

`int` и `int32` — это один и тот же тип.

`uint` и `uint32` — тоже один и тот же тип.

`float64` — основной вещественный тип.

`float32` принимается как синтаксис, но внутри используется общее представление float64.

`void` означает, что функция ничего не возвращает.

`unit` — это синоним `void`.

Отдельного типа `char` у меня нет. Если нужен символ, используется строка длины 1:

```
string c = "a";
```

---

## 3. int и int32

Целые литералы по умолчанию имеют тип `int`.

Пример:

```
int x = 42;
int32 y = 100;
```

`int` и `int32` внутри считаются одним каноническим типом.

То есть:

```
int x = 10;
int32 y = x;
```

это нормально.

Также поддерживаются разные формы записи целых чисел.

Обычная десятичная:

```
int a = 42;
```

Шестнадцатеричная:

```
int b = 0x2A;
```

Двоичная:

```
int c = 0b101010;
```

Все три примера дают число 42.

---

## 4. uint и uint32

`uint` — это беззнаковый целый тип.

Пример:

```
uint x = cast<uint>(10);
```

`uint` и `uint32` считаются одним типом.

Обычные целые литералы имеют тип `int`, поэтому для `uint` часто нужен явный cast.

Например:

```
uint a = cast<uint>(1);
uint b = cast<uint>(2);
```

Смешивать `int` и `uint` без явного cast нельзя.

Так лучше не писать:

```
int x = 10;
uint y = cast<uint>(5);
x + y;
```

Потому что это разные типы.

Нужно явно привести один тип к другому.

---

## 5. float64 и float32

`float64` — основной вещественный тип.

Пример:

```
float64 x = 3.14;
```

Есть запись с экспонентой:

```
float64 a = 1.5e2;
float64 b = 2.5E1;
```

Это значит:

```
1.5e2 = 150
2.5E1 = 25
```

Также есть специальные значения:

```
float64 a = inf;
float64 b = -inf;
float64 c = NaN;
```

Важно: в моей VM деление на ноль даёт runtime error даже для float64.

То есть так нельзя:

```
float64 x = 1.0 / 0.0;
```

А специальные значения нужно писать явно через литералы:

```
inf
-inf
NaN
```

`float32` у меня принимается, но внутри работает как общий вещественный тип.

---

## 6. bool

Тип `bool` имеет два значения:

```
true
false
```

Для bool есть логические операции:

```
!x
x && y
x || y
```

Пример:

```
bool ok = true;
bool result = ok && false;
```

`bool` нельзя использовать как число.

Так нельзя:

```
bool b = true;
int x = b + 1;
```

---

## 7. string

`string` — это строка.

Пример:

```
string s = "hello";
```

Строки можно складывать через `+`:

```
string a = "hello";
string b = " world";
string c = a + b;
```

Результат:

```
"hello world"
```

Строки можно сравнивать:

```
a == b
a != b
```

Сравнение идёт по содержимому.

Также для строки работает `len`:

```
int n = len("hello");
```

---

## 8. void и unit

`void` означает, что функция ничего не возвращает.

Пример:

```
void printHello() {
    print("hello\n");
    return;
}
```

`unit` — это синоним `void`.

То есть можно воспринимать:

```
unit
```

как другое имя для:

```
void
```

В функции `void` / `unit` можно писать `return;` без значения.

Но нельзя возвращать значение:

```
void f() {
    return 10;
}
```

Это ошибка.

---

## 9. Массивы

Массивы имеют фиксированный размер.

Пример:

```
int[3] a = [1, 2, 3];
```

Размер является частью типа.

То есть:

```
int[3]
```

и

```
int[4]
```

это разные типы.

С массивами можно делать:

```
a[i]
a[i] = value
len(a)
a == b
a != b
```

Пример:

```
int[3] a = [1, 2, 3];

print(a[0], "\n");

a[1] = 10;

print(len(a), "\n");
```

Если выйти за границы массива, будет runtime error:

```
int[2] a = [10, 20];
print(a[2]);
```

Индекс 2 неправильный, потому что есть только индексы 0 и 1.

---

## 10. Массивы разных типов

Сначала массивы были проще, но после доработки поддерживаются массивы разных типов:

```
int[3]
uint[3]
float64[2]
bool[2]
string[2]
Point[2]
```

Например:

```
uint[3] flags = [cast<uint>(1), cast<uint>(2), cast<uint>(4)];

float64[2] limits = [0.0, 100.0];

string[2] names = ["Almas", "Bob"];
```

Массивы структур тоже поддерживаются:

```
struct Point {
    int x;
    int y;
}

Point[2] points = [
    Point { x: 1, y: 2 },
    Point { x: 3, y: 4 }
];
```

---

## 11. Структуры

Структура — это пользовательский тип с полями.

Пример:

```
struct Point {
    int x;
    int y;
}
```

Создание значения структуры:

```
Point p = Point { x: 10, y: 20 };
```

Доступ к полю:

```
p.x
```

Изменение поля:

```
p.x = 30;
```

Сравнение:

```
p == q
p != q
```

Структуры сравниваются по полям.

Типы структур номинальные.

Это значит, что две структуры совместимы только если у них одно и то же имя типа.

Например:

```
struct A {
    int x;
}

struct B {
    int x;
}
```

`A` и `B` — разные типы, даже если поля одинаковые.

---

## 12. Массивы внутри структур

После доработок структура может содержать массивы.

Пример:

```
struct Group {
    string title;
    int[3] grades;
    string[2] names;
}
```

Использование:

```
Group g = Group {
    title: "VM group",
    grades: [90, 80, 70],
    names: ["Almas", "Bob"]
};
```

Можно читать элемент массива внутри структуры:

```
print(g.grades[0], "\n");
```

Можно менять элемент:

```
g.grades[1] = 100;
```

Это затрагивает сразу несколько частей проекта: структуры, массивы, field access, index access и semantic checking.

---

## 13. Type aliases

`type alias` задаёт другое имя существующему типу.

Пример:

```
type Meters = int;

int main() {
    Meters distance = 100;
    int x = distance;
    return x;
}
```

`Meters` не создаёт новый отдельный тип.

Это просто другое имя для `int`.

Поэтому `Meters` и `int` совместимы.

---

## 14. var type inference

`var` позволяет не писать тип переменной явно.

Пример:

```
var x = 10;
var y = 2.5;
var s = "hello";
var b = true;
```

Компилятор сам выводит тип по правой части:

```
x -> int
y -> float64
s -> string
b -> bool
```

Но после вывода типа переменная всё равно статически типизирована.

То есть нельзя сделать так:

```
var x = 10;
x = "hello";
```

Потому что `x` уже имеет тип `int`.

---

## 15. let и мутабельность

`let` создаёт неизменяемую переменную.

Пример:

```
let x = 10;
```

После этого нельзя писать:

```
x = 20;
```

Это semantic error.

Обычные объявления и `var` создают изменяемые переменные:

```
int x = 10;
x = 20;

var y = 5;
y = 6;
```

Мутабельность — это свойство переменной, а не типа.

---

## 16. cast

`cast` — это явное приведение типа.

Синтаксис:

```
cast<T>(expr)
```

Примеры:

```
int x = cast<int>(3.5);
uint u = cast<uint>(10);
float64 f = cast<float64>(x);
```

cast нужен, когда компилятор сам не должен неявно менять тип.

Например `int` и `uint` нельзя смешивать просто так. Нужно явно написать cast.

---

## 17. Generic-функции

В проекте есть generic-функции.

Пример:

```
T id<T>(T x) {
    return x;
}
```

Можно вызвать с явным типом:

```
int a = id<int>(42);
```

Можно с автовыводом:

```
string s = id("hello");
```

В этом случае компилятор сам понимает, что:

```
T = string
```

Также можно использовать generic-функции с несколькими параметрами типов, если типы можно вывести из аргументов или они указаны явно.

Например:

```
T choose<T>(bool cond, T a, T b) {
    if (cond) {
        return a;
    }

    return b;
}

int x = choose(true, 10, 20);
```

Тут `T = int`.

А вот так нельзя:

```
int x = choose(true, 10, "bad");
```

Потому что один аргумент говорит, что `T = int`, а другой говорит, что `T = string`.

Generic structs у меня не реализованы.

Важно: реализация generic ближе к type erasure.

То есть типы проверяются на этапе semantic analysis, но отдельная копия функции под каждый тип не создаётся.

Во время выполнения VM работает с обычным универсальным `Value`.

---

## 18. Перегрузка функций

В проекте есть перегрузка функций.

Можно написать несколько функций с одним именем, но разными параметрами:

```
int describe(int x) {
    return 1;
}

int describe(string x) {
    return 2;
}
```

Вызов:

```
describe(10)
```

выберет функцию для `int`.

Вызов:

```
describe("hello")
```

выберет функцию для `string`.

Если две функции имеют одинаковые параметры, это ошибка:

```
int f(int x) {
    return 1;
}

int f(int x) {
    return 2;
}
```

Semantic analyzer выбирает нужную перегрузку и записывает в AST конкретный functionIndex.

---

## 19. Перегрузка с неявными приведениями

Ещё есть перегрузка с неявными приведениями.

Если точной перегрузки нет, компилятор может попробовать безопасное числовое преобразование.

Поддерживаются:

```
int  -> float64
uint -> float64
```

Пример:

```
int onlyFloat(float64 x) {
    return 10;
}

int main() {
    return onlyFloat(42);
}
```

Тут передан `int`, а функция принимает `float64`.

Компилятор сам вставляет приведение.

Но если есть точное совпадение, оно всегда важнее.

Пример:

```
int f(int x) {
    return 1;
}

int f(float64 x) {
    return 2;
}

int main() {
    return f(42);
}
```

Выберется `f(int)`, а не `f(float64)`.

---

## 20. Meta-функции

Есть встроенные meta-функции:

```
sizeof(expr)
typeid(expr)
typeof(expr)
```

`sizeof` возвращает размер типа в байтах.

Пример:

```
sizeof(42)
```

даёт 4.

`typeid` возвращает строку с именем типа:

```
typeid(42)
```

даёт:

```
"int"
```

`typeof` в текущей реализации по смыслу похож на `typeid`.

Например:

```
typeof("hello")
```

даёт:

```
"string"
```

Эти функции полезны для проверки работы системы типов и для демонстрации на защите.

---

## 21. Битовые операции

Битовые операции работают для `int` и `uint`.

Операции:

```
&
|
^
~
<<
>>
```

Пример:

```
int a = 6 & 3;
int b = 6 | 3;
int c = 6 ^ 3;
int d = 1 << 4;
```

Для `&`, `|`, `^` типы должны совпадать.

То есть можно:

```
int & int
uint & uint
```

Но нельзя смешивать `int` и `uint` без cast.

Так нельзя:

```
int x = 1;
uint y = cast<uint>(2);
x & y;
```

Нужно привести типы явно.

`~` работает для `int` и `uint`.

Сдвиги:

```
a << n
a >> n
```

Левый операнд должен быть `int` или `uint`.

Правый операнд тоже должен быть `int` или `uint`.

Если сдвиг отрицательный или сдвиг больше либо равен 32, будет runtime error:

```
runtime error: invalid shift count at line N
```

---

## 22. Сравнение типов и совместимость

В целом язык строгий.

Если типы не совпадают, нужно явно делать `cast`.

Например нельзя просто сложить `int` и `uint`.

Нельзя присвоить `string` в `int`.

Нельзя передать `string` туда, где функция ждёт `int`.

Исключение — перегрузка функций с неявным приведением, где разрешены:

```
int  -> float64
uint -> float64
```

Но это используется именно при выборе перегрузки, а не везде подряд.

---

## 23. Value semantics

В языке используется value semantics.

Это значит, что значения ведут себя как значения, а не как указатели для пользователя языка.

Пример:

```
int a = 10;
int b = a;
b = 20;
```

После этого `a` остаётся 10.

Для строк, массивов и структур внутри VM могут использоваться runtime-объекты, но на уровне языка они должны восприниматься как значения.

---

## 24. Runtime-проверки

Не все ошибки можно поймать на этапе semantic analysis.

Например индекс массива может быть вычислен во время выполнения:

```
int i = input_number;
a[i]
```

Поэтому есть runtime-проверки:

```
выход за границы массива
деление на ноль
modulo by zero
неправильный сдвиг
assert failure
panic
```

Ошибки runtime выводятся примерно так:

```
runtime error: message at line N
```

---

## 25. Что не реализовано

Не реализовано:

```
отдельный char
generic structs
nullable types
lambdas
first-class functions
pointers
полноценная модульная система
overloaded operators
```

Также ASM backend поддерживает не весь язык. Основной backend — это bytecode VM.


В MiniLangVM статическая строгая типизация. Тип каждого выражения определяется на этапе semantic analysis. Если типы несовместимы, байткод не генерируется.
```

Про типы:

```
Есть int/int32, uint/uint32, float64/float32, bool, string, void/unit, fixed-size arrays, structs, type aliases и generic parameters.
```

Про массивы:

```
Массивы имеют фиксированный размер, размер входит в тип. Поддерживаются массивы int, uint, float64, bool, string и struct, в том числе массивы как поля структур.
```

Про generic:

```
Generic-функции проверяются на этапе semantic analysis. Есть явные type arguments и автовывод. Реализация ближе к type erasure, потому что отдельные копии функций под каждый тип не создаются.
```

Про runtime:

```
Semantic analyzer ловит ошибки типов до запуска, а VM ловит ошибки, которые зависят от runtime-значений: деление на ноль, выход массива за границы, неправильный сдвиг, assert и panic.
```
